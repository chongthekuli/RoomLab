# RoomLAB Regression Index

**What this is.** One row per shipped code-bug. Each row links the user-visible symptom to the guarding test that would have caught it. If a fix shipped without a test, the row is `UNGUARDED` and routes to Sam (qa-engineer) for a spec.
**Who maintains it.** Theo Halvorsen (regression-curator). Same-PR rule: every bug-fix commit must add or extend a regression test in the SAME commit; no exceptions, no "next sprint."
**Legend.**  `GUARDED` = a real test asserts the bug stays fixed. `UNGUARDED` = fix shipped without a test; needs a spec. `OBSOLETE` = the code path that hosted the bug has been removed (must link the removing commit).
**How to add a row.** Append the next sequential `RL-NNN`. Fill every column. Use a real commit-sha (short, 7 chars). For `GUARDED` rows, give file + test name verbatim (e.g. `tests/spl.test.mjs :: "Back-pivot: adjacent cabinets share their back edge"`). For `UNGUARDED` rows, leave the test column empty and add a TODO note.

> **Scope note.** This index tracks **code bugs** only.
